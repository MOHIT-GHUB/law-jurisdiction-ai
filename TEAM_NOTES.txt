================================================================================
  SECTION 1 — BUGS THAT WERE FOUND AND FIXED (understand these)
================================================================================

BUG 1 — Circular Import (FIXED)
  WHAT: graph.py imported agents, agents imported AgentState from graph.py
        → Python raises ImportError on startup. App would not start at all.
  FIX:  AgentState moved to agents/state.py (no internal imports)
        All agents now import from state.py. graph.py imports from state.py.
  RULE: Never move AgentState back into graph.py or any agent file.

BUG 2 — Broken Fan-Out (FIXED)
  WHAT: state_law_agent and case_law_agent had no edges in the graph.
        Only federal_law_agent was wired. The other two would never run.
  FIX:  fan_out_research node added. Uses asyncio.gather() to run all
        3 agents truly in parallel. Much faster than sequential.

BUG 3 — stream_callback Not Serializable (FIXED)
  WHAT: stream_callback (async function) was in AgentState TypedDict.
        MemorySaver checkpointer tries to serialize state → crashes.
  FIX:  Removed from AgentState. Now passed via LangGraph config dict.
        Agents read it from: config.get("configurable", {}).get("stream_callback")
  NOTE: Agents currently have a placeholder for this. The WebSocket handler
        must wire it up correctly (see Section 2 below).

BUG 4 — Missing pdf_export.py (NOT FIXED — TEAM MUST CREATE)
  WHAT: conversations.py imports from app.utils.pdf_export which doesn't exist.
        App will fail to import/start until this file is created.
  FIX:  See Section 2 — MISSING FILES.

  SECTION 3 — FRONTEND (React)
================================================================================

Recommended stack: React + TypeScript + Tailwind CSS + shadcn/ui
Why: Fastest to build a polished UI. Judges see frontend first.

Key components needed:

  AuthPage.tsx        — Login/Signup form
                        Simple. Ask AI: "write a React login/signup form
                        that calls POST /auth/login and stores the JWT"

  Sidebar.tsx         — Conversation list (like ChatGPT)
                        Calls GET /conversations/, shows title + date

  ChatWindow.tsx      — Main chat area                          ← HARDEST
                        Renders messages with markdown
                        Shows typing indicator during streaming
                        Handles WebSocket connection

  ScoreCard.tsx       — Case Strength Score visual              ← WIN FACTOR
                        Animated circular progress bar 0-100
                        Color: red/yellow/green based on score
                        This is what impresses judges most in the demo

  LawyerCard.tsx      — Shows referred attorneys with ratings

  ExportButton.tsx    — Button that triggers PDF download
                        Calls GET /conversations/{id}/export-pdf

WebSocket message protocol (agree on this with the team):
  Client → Server: { type: "message", content: "I was fired yesterday..." }
  Server → Client: { type: "token", content: "I understand..." }   (streaming)
  Server → Client: { type: "intake_complete" }                      (research starting)
  Server → Client: { type: "score", value: 74 }                     (final score)
  Server → Client: { type: "done" }                                 (complete)
  Server → Client: { type: "error", message: "..." }                (guard/error)

Effort estimate: 2 days for a decent frontend. Start early.
Use v0.dev (Vercel's AI) to generate initial component skeletons quickly.
